# Design System Reference

When building UI for this project, follow these principles rigorously. The goal is output that looks like it was crafted by a senior designer at Linear, Vercel, or Stripe — not generated by AI.

## What makes AI-generated UI obvious (AVOID ALL OF THESE)

- Generic hero sections with "Welcome to [Product]" and a gradient background
- Emoji as icons (use SVG icons from Lucide, Heroicons, or custom)
- Cards with identical padding, identical border-radius, identical shadows
- Overly symmetrical layouts — real designs have intentional asymmetry
- Rainbow gradient buttons or text
- Generic stock-photo-style placeholder content
- "Lorem ipsum" — always use realistic content
- Too many colors — professional sites use 1-2 accent colors max
- Equal spacing between all elements — use a clear spacing scale
- Rounded corners on everything at the same radius
- Drop shadows on everything
- Centered text for everything — left-align body text
- Excessive whitespace with no purpose
- Generic sans-serif at one size — use clear typographic hierarchy

## Typography

Use Inter, or the project's chosen font. Follow this scale strictly:

```
Display:  48-72px, weight 700-800, letter-spacing: -0.025em, line-height: 1.1
H1:       36-48px, weight 700, letter-spacing: -0.025em, line-height: 1.15
H2:       28-32px, weight 600, letter-spacing: -0.02em, line-height: 1.2
H3:       20-24px, weight 600, letter-spacing: -0.01em, line-height: 1.3
Body:     15-16px, weight 400, letter-spacing: 0, line-height: 1.6
Small:    13-14px, weight 400, letter-spacing: 0, line-height: 1.5
Caption:  11-12px, weight 500, letter-spacing: 0.02em, line-height: 1.4
```

Key rules:
- Negative letter-spacing on headings (makes them feel tighter, more premium)
- Positive letter-spacing on captions/labels (improves readability at small sizes)
- Max line length for body text: 65-75 characters (~600px)
- Use font-weight contrast, not size contrast, for subtle hierarchy
- Headings: 700. Subheadings: 500-600. Body: 400. NEVER use 300 for body text.

## Spacing

Use a 4px base grid. Common spacing values:

```
4px   — micro gaps (icon to label)
8px   — tight element spacing
12px  — related element groups
16px  — standard padding
20px  — comfortable padding
24px  — section internal spacing
32px  — between content blocks
48px  — between major sections
64px  — page section breaks
96px  — hero/major section padding
```

Key rules:
- Increase spacing as you go up the hierarchy (more space between sections than between paragraphs)
- Group related elements with less space, separate unrelated elements with more
- Padding inside containers: 16-24px. Never less than 12px.
- Vertical rhythm: use consistent multiples

## Color

Use a monochromatic or analogous palette. Never more than 3 colors:

```
Background:     Very dark (#0a0a0a) or very light (#fafafa) — never pure black or white
Surface:        One step lighter/darker than background
Text primary:   High contrast but not max (#e8e8e8 dark / #1a1a1a light)
Text secondary: Muted (#8b8b8b dark / #6b6b6b light)
Text tertiary:  Dim, for labels (#525252 dark / #999 light)
Accent:         ONE color for CTAs and interactive elements
Success:        #34d399 (emerald, not lime green)
Error:          #f87171 (rose, not red)
Warning:        #fbbf24 (amber, not yellow)
Border:         Barely visible (#1e1e1e dark / #e5e5e5 light)
```

Key rules:
- Maximum 2 accent colors in the entire app
- Use opacity variations of your accent for hover/active states, not new colors
- Borders should be subtle — 1px, low contrast
- Never use pure black (#000) for text on white — use #1a1a1a or #111
- Shadows: use very subtle shadows. 0 2px 8px rgba(0,0,0,0.08) light mode. Avoid in dark mode.

## Layout Patterns (from Linear, Vercel, Stripe)

### Hero sections
- Headline: 48-64px, bold, tight letter-spacing
- Subhead: 18-20px, regular weight, muted color, max 2 lines
- CTA: ONE primary button, optionally one ghost/secondary
- Trust signals below: small logos or stats, very muted
- NO background gradients unless extremely subtle (opacity < 0.05)

### Feature sections
- 2-3 column grid (never 4+ on desktop)
- Each feature: icon (24px, stroke style) + heading (18px, semibold) + description (14-15px, muted)
- Generous gap between features (32-48px)
- Icons should be monochrome stroke style, not filled, not emoji

### Pricing tables
- Max 3 tiers
- Highlight the recommended tier with a subtle border or background change, NOT a gradient
- Price: large and bold, period is small and muted
- Feature list: checkmarks in muted green, text in secondary color
- Equal column widths

### Testimonials
- Quote text: slightly larger than body, italic or regular weight
- Attribution: small, muted, with role/company
- NO star ratings unless product requires them
- Avatar: small circle (32-40px), or just text attribution

### Navigation
- Height: 56-64px
- Logo left, nav center or left, CTA right
- Links: 14px, weight 500, muted color, darken on hover
- Mobile: hamburger or bottom nav, never both
- Sticky: add subtle border-bottom or shadow on scroll

### Footer
- Darker than page background
- Multi-column links, muted color
- Small text: 13px
- Bottom bar with copyright: very muted

## Components

### Buttons
- Primary: solid background, white text, 8px radius, 12-14px font, 500-600 weight
- Secondary: border only, no fill, muted text
- Height: 36-40px for default, 32px for small, 44-48px for large
- Horizontal padding: 16-24px
- NEVER use gradients on buttons
- Hover: darken by 8-10%, subtle transform: translateY(-1px)
- Active: darken more, translateY(0)
- Disabled: 40% opacity, no hover effects

### Inputs
- Height: 40-44px
- Border: 1px solid, subtle color
- Border-radius: 8px
- Padding: 12-16px horizontal
- Focus: accent color border, optional subtle ring (0 0 0 3px accent/10%)
- Label above, not inside (floating labels are outdated)
- Placeholder: tertiary color, shorter than label

### Cards
- Background: surface color (one step from background)
- Border: 1px solid, barely visible
- Border-radius: 8-12px (pick ONE value, use it everywhere)
- Padding: 20-24px
- Hover: slightly elevate border contrast, optional translateY(-2px)
- NO shadows in dark mode. Subtle in light mode only.

### Modals
- Max width: 480px for dialogs, 640px for forms, 900px+ for complex
- Border-radius: 12-16px
- Overlay: rgba(0,0,0,0.6) with backdrop-filter: blur(8px)
- Enter animation: scale(0.96) + translateY(8px) to scale(1) + translateY(0)
- Close: X button top-right, or click overlay

## Animations (from Apple HIG / Framer Motion)

- Duration: 150-300ms for micro-interactions, 300-500ms for page transitions
- Easing: cubic-bezier(0.22, 1, 0.36, 1) for entrances (smooth deceleration)
- Easing: cubic-bezier(0.34, 1.56, 0.64, 1) for bouncy/playful (buttons, toasts)
- NEVER use linear easing for UI elements
- Hover effects: 150ms
- Button press: scale(0.97), 60ms, spring
- Page transitions: fade + translateY(8px), 250ms
- Stagger children: 30-50ms delay between items

## Mobile Responsive

- Breakpoints: 640px (mobile), 768px (tablet), 1024px (desktop), 1280px (wide)
- Mobile-first: start with mobile layout, add complexity for larger screens
- Touch targets: minimum 44px height
- Reduce heading sizes by ~25% on mobile
- Stack columns on mobile, never side-scroll
- Bottom padding on mobile: at least 32px (for safe area)
